# 个人规则 (Personal Rules)

## 1. 核心环境与背景 (Core Environment & Context)
- **对话语言**: 请始终使用 **简体中文** 与我交流。
- **操作系统**: 我的操作系统是 **Windows**。
- **命令偏好**: 在提供任何命令行操作时，请优先考虑 Windows 环境。例如，使用 **PowerShell** 或 **CMD** 语法（如 `dir`, `copy`），而不是 Linux/macOS 的 Bash 语法（如 `ls`, `cp`）。如果必须提供跨平台命令，请同时注明 Windows 下的对应命令。

## 2. 通用交流风格
- **风格**: 回答应专业、简洁、切中要点。在提供代码示例后，请附上关键部分的解释。

## 3. 代码生成规则
- **默认语言**: 所有代码生成任务，请默认使用 **TypeScript**。除非我明确要求，否则不要使用 JavaScript。
- **代码注释**:
    - 为所有非匿名的函数或方法添加 **JSDoc** 格式的注释，说明其功能、参数 (`@param`) 和返回值 (`@returns`)。
    - 对于复杂的逻辑或算法，请在关键步骤上方添加行内注释进行解释。
- **错误处理**: 在生成涉及到 I/O 操作（如文件读写、API 请求、数据库操作）的代码时，请默认包含 `try...catch` 结构进行错误处理，并返回有意义的错误信息。
- **包管理器**: 在所有涉及到安装依赖的命令中，请优先使用 `pnpm`。例如，使用 `pnpm install <package>` 而不是 `npm install <package>`。
- **质量与风格**: 生成的所有代码，必须同时满足：
    1.  **ESLint 推荐的最佳实践** (例如，无未使用的变量、使用 `===` 代替 `==`、遵循 React Hooks 规则等)。
    2.  **Prettier 的默认格式化规则** (例如，统一的缩进、引号、分号、行长等)。
